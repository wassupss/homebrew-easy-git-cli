# Easy Git 사용 예제

## 시작하기

### 첫 실행

```bash
# 프로젝트 디렉토리로 이동
cd your-project

# Easy Git 실행
easy-git
# 또는
eg
```

## 일반적인 작업 흐름

### 🎯 시나리오 1: 기본 커밋 & 푸시

```
1. Easy Git 실행
   $ eg

2. "📊 상태 확인" 선택
   → 현재 변경된 파일들 확인

3. "➕ 파일 추가" 선택
   → "모든 파일 추가" 또는 "특정 파일 선택"

4. "💾 커밋" 선택
   → 커밋 메시지 입력: "feat: 새로운 기능 추가"

5. "⬆️ 푸시" 선택
   → 확인 후 푸시 완료!
```

### 🎯 시나리오 2: 새 기능 개발 (브랜치)

```
1. "🌿 브랜치 관리" 선택

2. "➕ 새 브랜치 생성" 선택
   → 브랜치 이름 입력: "feature-login"
   → 자동으로 새 브랜치로 전환됨

3. 코드 작성...

4. 변경사항 커밋 (시나리오 1 참고)

5. "⬆️ 푸시" 선택
   → feature-login 브랜치가 원격에 푸시됨

6. 작업 완료 후 main 브랜치로 전환
   "🌿 브랜치 관리" → "🔀 브랜치 전환" → "main" 선택
```

### 🎯 시나리오 3: 긴급 수정 (Stash 활용)

```
1. 현재 작업 중 (변경사항이 있지만 커밋하지 않음)

2. 긴급 버그 수정 요청이 옴

3. "📦 Stash 관리" 선택
   → "💾 현재 변경사항 저장" 선택
   → 메시지 입력: "작업 중인 로그인 기능"

4. "🌿 브랜치 관리" → main 브랜치로 전환
   (작업 디렉토리가 깨끗해짐)

5. 버그 수정 후 커밋 & 푸시

6. 다시 feature 브랜치로 전환

7. "📦 Stash 관리" → "♻️ 최근 Stash 복원"
   → 이전 작업 내용 복원!
```

### 🎯 시나리오 4: 팀원 코드 가져오기

```
1. "⬇️ 풀" 선택
   → 현재 브랜치의 최신 코드 가져오기

2. 충돌이 발생한 경우:
   - "📊 상태 확인"에서 충돌 파일 확인
   - 수동으로 충돌 해결
   - "➕ 파일 추가"로 해결된 파일 추가
   - "💾 커밋"으로 머지 커밋 생성
```

### 🎯 시나리오 5: 커밋 히스토리 확인

```
1. "📜 로그 보기" 선택

2. 보고 싶은 커밋 개수 선택
   - 최근 5개
   - 최근 10개
   - 최근 20개
   - 최근 50개

3. 커밋 정보 확인:
   - 커밋 해시
   - 커밋 메시지
   - 작성자
   - 날짜
```

## 고급 사용법

### 여러 파일 선택적으로 추가

```
1. "➕ 파일 추가" → "특정 파일 선택"

2. 스페이스바로 원하는 파일 선택
   [x] src/index.ts
   [ ] src/test.ts
   [x] README.md

3. 엔터로 확인
```

### 브랜치 삭제

```
1. "🌿 브랜치 관리" → "🗑️ 브랜치 삭제"

2. 삭제할 브랜치 선택
   (현재 브랜치는 선택 불가)

3. 강제 삭제 여부 확인
   - 머지되지 않은 변경사항이 있으면 경고
```

### Stash 목록 관리

```
1. "📦 Stash 관리" → "📋 Stash 목록 보기"

2. 저장된 모든 stash 확인
   0. stash@{0}: WIP on main: 작업 중
   1. stash@{1}: feat: 로그인 작업 중
   2. stash@{2}: fix: 긴급 수정 전 백업

3. 특정 stash 삭제
   "🗑️ Stash 삭제" → 인덱스 선택
```

## 키보드 단축키

### 메뉴 네비게이션

- `↑/↓`: 위/아래 이동
- `Enter`: 선택
- `Ctrl+C`: 종료

### 체크박스 (파일 선택시)

- `Space`: 선택/해제
- `a`: 전체 선택 (inquirer 기본)
- `i`: 선택 반전 (inquirer 기본)

## 문제 해결

### "Git 저장소가 아닙니다" 에러

```
해결: "새 Git 저장소 초기화" 선택
또는 올바른 프로젝트 디렉토리로 이동
```

### Push 실패 - Upstream 브랜치 없음

```
Easy Git이 자동으로 upstream 설정 제안
"원격 브랜치를 설정하고 푸시하시겠습니까?" → 예
```

### 충돌 발생시

```
1. "📊 상태 확인"에서 충돌 파일 확인 (⚠️ 마젠타색)
2. 텍스트 에디터에서 수동 해결
3. "➕ 파일 추가"
4. "💾 커밋"으로 머지 완료
```

### Stash 복원시 충돌

```
1. 충돌 파일 수동 해결
2. "➕ 파일 추가"
3. "💾 커밋"하지 않고 계속 작업
```

## 팁과 트릭

### 💡 빠른 작업 흐름

```
상태 확인 → 추가 → 커밋 → 푸시
(S → A → C → P)
```

### 💡 커밋 전 항상 상태 확인

```
어떤 파일이 변경되었는지 확인하는 습관
예상치 못한 파일이 포함되는 것 방지
```

### 💡 의미있는 커밋 메시지

```
좋은 예:
- "feat: 사용자 로그인 기능 추가"
- "fix: 회원가입시 이메일 검증 버그 수정"
- "docs: README에 설치 방법 추가"

나쁜 예:
- "수정"
- "커밋"
- "ㅇㅇ"
```

### 💡 자주 푸시하기

```
작업이 끝날 때마다 푸시하면:
- 코드 백업
- 팀원과 공유
- 작업 내역 추적
```

### 💡 브랜치 전략

```
main: 안정적인 코드
feature/*: 새 기능 개발
bugfix/*: 버그 수정
hotfix/*: 긴급 수정
```
